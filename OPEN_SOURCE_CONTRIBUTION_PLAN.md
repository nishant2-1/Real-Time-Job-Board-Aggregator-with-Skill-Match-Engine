# Open Source Contribution Plan (Honest Activity)

This plan is for real, reviewable contributions only.

## Rules

- No auto-commit workflows for graph coloring.
- Every green square should map to meaningful work.
- Prefer PRs, issues, docs, tests, refactors, and bug fixes.

## Weekly Rhythm

- Monday: pick 1 issue and reproduce it.
- Tuesday: small fix or docs improvement PR.
- Wednesday: tests or refactor PR.
- Thursday: feature or enhancement PR.
- Friday: review, comments, and follow-up commits.
- Weekend: optional learning PR or README cleanup.

## Intensity Levels (Natural Light/Dark Mix)

- Light day: 1 small commit (docs, typo fix, test adjustment).
- Medium day: 2-4 commits (bug fix + tests + review tweaks).
- Dark day: 5+ commits (feature branch with iterative commits).

## Contribution Types To Rotate

- Code: bug fixes, features, refactors
- Quality: test coverage, flaky test fixes, CI fixes
- Docs: API docs, setup docs, migration notes
- Community: issue triage, PR reviews, reproducible bug reports

## Anti-Pattern To Avoid

- Empty commits only for chart activity.
- Meaningless whitespace-only churn.
- Automated fake activity scripts.

## Monthly Review

- Count merged PRs (not just commits).
- Highlight user-visible improvements.
- Keep a short changelog of impact.
